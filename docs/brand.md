## 5) Mission

> Agents should be accountable.  
> Not just autonomous.

Unbox เชื่อว่าโลกที่ agent ทำงานแทนมนุษย์ได้ ต้องเป็นโลกที่มนุษย์ยังมองเห็น:
- เห็นการตัดสินใจ
- เห็นเหตุผล
- เห็นผลที่ตามมา

Unbox ไม่ได้หยุด agent ไม่ให้ทำงาน แต่ทำให้ผู้ใช้เห็นว่า agent กำลังทำอะไร และทำไม

## 8) Tone of voice rules

### Do

- ตรงไปตรงมา ไม่ขาย hype
- พูดถึง agent เหมือน entity ที่มีเหตุผล
- technical แต่อ่านรู้เรื่อง
- numbers-first
- active voice

### Don't

- ห้ามใช้คำ superlative เช่น `best`, `first`, `revolutionary`, `unprecedented`
- ห้ามใช้ jargon โดยไม่อธิบาย
- ห้าม copy ที่โอเวอร์เคลม

### Language rules (5)

1. ตัวเลขมาก่อน  
   - ใช้: `312 decisions, 94% accuracy`
2. ใช้ active voice เท่านั้น  
   - ใช้: `The agent blocked...`
3. ไม่มีคำ superlative  
4. พูดความไม่แน่นอนตรงๆ  
   - ใช้: `We estimate...`, `Data suggests...`, `We don't know yet`
5. Agent ใช้คำว่า `it`  
   - ไม่ blur เป็น "the system"

## 13) Usage policy for UX and content

When writing UX copy or product messaging:

1. Follow this file first, then `Unbox_PRD_v0.1.md`.
2. Any new tagline/headline must pass:
   - numbers-first check
   - no-hype check
   - active-voice check
3. Critical states (error/risk/uncertainty) must be explicit, factual, and actionable.
